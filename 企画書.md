# Inbound UGC Analysis 企画書（観光UGCインバウンド分析）

**一行サマリ：** 観光地名を入れると、YouTube上のUGCを「誰にどの話題が効いているか」で可視化し、自治体担当者がインバウンド施策の手がかりを得られる意思決定支援ツール。

> 本書は、課題に対する自分の思考プロセス（着眼点・取捨選択・アプローチ・工夫）をまとめた企画書です。動くプロトタイプを別途実装しており（本リポジトリ）、本書はその設計判断の背景を説明するものです。

---

## 1. 課題の捉え方（着眼点）

与えられたテーマは「SNS上のUGCを分析し、インバウンド誘致のプロモーション戦略を立てるシステム」でした。最初に立ち止まって考えたのは、**「このテーマで一番難しいのは“分析アルゴリズム”ではなく“データが本当に取れるか”だ」** という点です。

UGC分析というと解析ロジックに目が行きがちですが、各SNSのAPIは年々厳しくなっており、「分析する以前に、第三者の投稿をどう合法的に・現実的なコストで取得するか」が成否を分けます。ここを曖昧にしたまま機能を設計すると、絵に描いた餅になります。そこで私は**「データ取得の実現性」を企画の出発点**に据えました。

あわせて、当初テーマには曖昧な点が複数あると気づきました。

- 「インプレッション分析」とあるが、インプレッションは投稿者本人にしか開示されず、第三者UGCでは原則取得できない
- 最終目的は「インバウンド（訪日外国人）」なのに、分析対象が「UGC一般」になりがちで、外国人の動機把握に直結しない
- 「エンゲージメント率」「トレンド」などの指標が未定義で、このままでは数値が比較不能

**「要件をそのまま実装する」のではなく、「要件の穴を埋めてから実装する」** のが、この課題に対する私の基本姿勢です。

## 2. 既存知見の調査と取捨選択

「実現可能性」を判断するため、主要SNSのAPIの取得条件・コスト・制約を調べ、観光UGC収集という用途で比較しました。

| SNS | 費用 | 取得しやすさ | 判断 |
|---|---|---|---|
| YouTube Data API v3 | 無料（1日1万ユニット） | ◎ | **採用** |
| Google Places（クチコミ） | ほぼ無料 | ◎ | 将来併用候補 |
| Instagram Graph API | 無料だが審査・FB連携必須、HT検索 週30件 | △ | 拡張フェーズ |
| X (Twitter) API | 従量課金・高額（無料枠廃止） | △ | 予算次第 |
| TikTok Research API | 非営利/学術限定・商用除外 | ✕ | MVPでは除外 |

**取捨選択の結論：MVPはYouTube単独。** 理由は、(1) 無料で審査が軽く着手できる、(2) 観光動画・レビューが豊富で訪日客の発信も多い、(3) 同じコードが本番APIキーでそのまま動く、の3点。X・TikTokは「やりたくてもデータが取れない/高い」ため、最初から外す勇気を持つことが実現可能性の設計だと考えました。

なお、課題本文ではSNSは **「自由選択可（Instagram, TikTok, X, YouTube）」** と明記されています。したがってYouTube単独への絞り込みは、課題が許容する範囲内での合理的な意思決定です。「どれを選んでもよい」という条件に対し、調査に基づいて最も実現可能性の高い媒体を根拠を持って1つ選んだ、という点を見ていただきたいです。

> 既存の類似サービス（SNS分析ツール）の多くが有料APIや独自クロールに依存している中で、「無料・合法・MVPで完結」できる経路としてYouTubeを選んだのが、私なりの取捨選択です。

## 3. アプローチ（どう設計したか）

調査を踏まえ、当初テーマの穴を埋める形で要件を再定義しました。主要な設計判断は次の通りです。

1. **指標の再定義** — 取得不可能なインプレッションは捨て、一次指標を「公開エンゲージメント＝いいね＋コメント」に置き換えた。エンゲージメント率の分母は登録者数に固定（後述の工夫）。
2. **インバウンド特化のセグメント** — UGCを「言語（主軸）」と「発信者の推定地域（従軸）」で分割し、市場ごとに評価する。これにより「日本人にウケる動画」と「狙う市場にウケる動画」を区別できる。
3. **スコープを絞る** — 「任意の観光地・全SNS・全市場」を一度に作らず、YouTube／1観光地／直近12ヶ月／コア機能に絞ったMVPで、まず実現性と有用性を検証する段階設計にした。

この「再定義 → 絞り込み → 検証」の流れ自体を、課題への向き合い方として提示したいと考えています。

## 4. 工夫した点

**(1) 「取れない指標」を正直に捨て、要件を成立させた**
インプレッション分析は一見もっともらしいですが、技術的に不可能です。ここを曖昧にせず「捨てて、代替指標に置き換える」判断をしたことが、実装が破綻しなかった最大の要因です。

**(2) セグメント分割で「曖昧な評価」を解いた**
当初テーマの課題「利用層・利用目的の相違による曖昧な評価」に対し、発信者を言語・地域で切り分け、グループ内で比較する設計で応えました。さらに、少数セルには「参考値」フラグを付け、地域は自己申告ベースの推定値だと明示することで、**数字を過信させない誠実さ**を担保しました。

**(3) 高反応パターンを lift（偏り倍率）で抽出**
単に「人気動画はこれ」ではなく、上位群に偏在する特徴（話題・尺・投稿時期）を全体分布と比較し、「上位群での割合 ÷ 全体での割合」で偏りを定量化しました。例：サンプルでは「ショート動画」が高反応群に3.4倍偏在、といった再現可能な示唆が出ます。

**(4) 検索の入口を多言語化し、実データの泥臭さに対処した**
テスト中に「富山」と「toyama」で結果が大きく違うことに気づきました。日本語名で検索すると日本語コンテンツに偏り、インバウンド分析として成立しません。そこで地名を各市場の言語表記でも検索・照合する多言語化を実装。任意の観光地に対応するためWikidataの自動取得を試みましたが、「別府」が同名の*姓*に化けるなど曖昧性で失敗。最終的に**Wikipediaの言語間リンク**＋「対象言語のリンクが揃う記事を選ぶ」方式で解決しました。実データに当たって初めて見える問題を、調べて作り直した過程そのものを評価いただきたいです。

**(5) 「取れない視聴者層」をコメントで近似した**
視聴者の属性（年齢・地域）は第三者UGCでは取得できません。そこで**コメントの言語構成を「視聴者側」の代理指標**にし、発信者言語と並べて比較しました。サンプルでは「コメントの約7割が外国語で、発信者構成以上に海外の関心が高い」という、発信者≠視聴者の補正が示唆として出ます。あわせて感情分析でネガティブ要因（混雑・価格など）の起点も拾います。

**(6) 表現揺れをLLMで吸収（ローカル・キー不要も両立）**
話題分類はキーワード一致だと多言語の言い回しを取りこぼします。LLMに差し替えて表現揺れを吸収しつつ、**Ollamaでローカル実行（キー不要）**にも対応。LLMが無ければキーワード/辞書へ自動フォールバックするので、誰でも動かせます。

## 5. 実現可能性の設計

- **コスト：** YouTube Data API v3は無料。1日1万ユニット。
- **法務：** 第三者UGCの扱いには規約・個人情報の論点があるため、MVP検証と並行して順次対応する位置づけに整理（最初から完璧を求めず、止めない設計）。
- **段階的拡張：** MVP（YouTube・コア機能）→ 拡張1（感情分析・トレンド・競合比較）→ 拡張2（他SNS・外部指標突合で成果測定）。

「何を今やらないか」を明確にしたことが、実現可能性の肝だと考えています。

## 6. 課題「成果物例」との対応

課題本文に挙げられた分析項目に対し、MVPで「実装したもの・取得制約で代替したもの・拡張に回したもの」を明示します。**全部を作り込むより、各項目に対する判断を示すこと**を重視しました。

| 課題の成果物例 | 本MVPでの対応 | 状態 |
|---|---|---|
| 主要キーワードの投稿トレンド分析 | 地名＋カテゴリ語で投稿を収集し話題分類 | 対応 |
| 主要キーワードの検索・閲覧トレンド分析 | 検索トレンドはGoogle Trends連携、閲覧トレンドは時系列分析で拡張予定 | 拡張 |
| 人気観光系アカウント・投稿の特徴分析 | 高反応動画の特徴を抽出（アカウント単位の集約は今後強化） | 概ね対応 |
| エンゲージメント率の高いコンテンツパターン特定 | lift（偏り倍率）で上位群の特徴を定量化 | 中核実装 |
| 温泉・秘境系コンテンツの**視聴者層**分析 | 視聴者属性はAPI非開示→**コメントの言語構成**を視聴者側の代理指標として実装（下記） | 実装（近似） |
| 国内外での関心度の違い | 言語/地域セグメント＋発信者言語×視聴者言語の比較で可視化 | 対応 |
| 競合観光地のプロモーション手法調査 | 複数地点の収集・相対比較として拡張フェーズで対応 | 拡張 |

**「視聴者層分析」について（重要な着眼点）：** インプレッションと同じく、第三者UGCの視聴者属性（年齢・性別・地域）はYouTube APIでは取得できません（自チャンネルのアナリティクスのみ開示）。そこで本MVPは、取れない視聴者層を装うのではなく、**コメントの言語構成を「視聴者側」の代理指標**として実装し、発信者言語と並べて比較しました。あわせて**コメントの感情分析**でネガティブ要因の起点も拾います。「取れないものは正直に近似する」という一貫した判断です。

## 7. プロトタイプ（実装したもの）

Python製のCLIツールとして実装し、観光地名を渡すとHTMLレポートを生成します。

```bash
python -m youtube_ugc.cli "高山" --sample --out report.html        # サンプルで即実行
export YOUTUBE_API_KEY="..." && python -m youtube_ugc.cli "別府" --auto-aliases  # 本番収集
```

主な機能は、(1) 多言語地名での収集（任意の観光地に対応）、(2) 言語×地域セグメント、(3) エンゲージメント評価と lift による高反応パターン抽出、(4) コメントからの視聴者言語推定と感情分析、(5) 話題分類のLLM化（Ollama対応・無ければ辞書）。レポートは「現状サマリ → 市場別セグメント → 視聴者層と感情 → 人気コンテンツ → 高反応パターン → 示唆」の流れで、非専門の担当者がそのまま読めることを意図しています。

分析ロジックは**ユニットテスト36件**で検証済みです（言語誤判定や正規化のバグも実装中に発見・修正）。詳細な設計判断は付録の要件定義書（`docs/`）にまとめていますが、提出物の主役はこの企画書とプロトタイプです。

## 9. 振り返り（自分なりの視点）

この課題で意識したのは、**「言われた通りに作る」のではなく「言われたことの前提を疑い、現実に着地させる」** ことです。インプレッションを捨てる、SNSを1つに絞る、地域は推定値だと明示する——いずれも華やかではありませんが、こうした地に足のついた判断の積み重ねが、実際に動くプロダクトと、信頼できる示唆につながると考えています。
